```markdown
## 今日迁移：[温度]°C → [日期]

### 准备（5分钟）
- [ ] 确认原 HTML 文件存在 `public/[温度]-c-to-f.html`
- [ ] 查看原文件 SEO（title、description）
- [ ] 选择参考模板

### 执行（25-35分钟）
- [ ] Step 1: 复制翻译文件 `cp locales/en/[参考模板].json locales/en/[新温度].json`
  - [ ] 修改温度值
  - [ ] **复制原 HTML 的 title 和 description**
- [ ] Step 2: 复制页面组件 `cp pages/[参考模板].tsx pages/[新温度].tsx`
  - [ ] 修改 celsius = [新温度]
  - [ ] 调整颜色主题（如需要）
- [ ] Step 3: AI 翻译 10 种语言
  - [ ] 重点审核 Hindi
- [ ] Step 4: 更新 `next.config.js` redirects
- [ ] Step 5: `npm run build` 验证

### 验证（5分钟）
- [ ] 构建成功，0 错误
- [ ] 标题与原 HTML 一致
- [ ] 10 个语言版本都生成

### 完成
- [ ] 标记本页面为已完成
- [ ] 记录实际耗时
```

## 💡 提高效率的建议

### 1. **批量准备（周末做）**
- 一次性为下周的 7 个页面准备好英文翻译文件
- 周一到周五只需执行翻译和构建

### 2. **使用脚本辅助**
```bash
# 创建迁移脚本
echo "开始迁移 $1°C"
cp locales/en/36-c-to-f.json "locales/en/$1-c-to-f.json"
sed -i "s/36/$1/g" "locales/en/$1-c-to-f.json"
# ... 其他自动化
```

### 3. **分工策略**
- **周一-周三**：体温相关（使用相同模板，快速复制）
- **周四-周五**：烹饪/天气（需要调整内容）
- **周末**：审核 Hindi 翻译，准备下周文件
